# Frontend design system (tokens.css)

- `frontend/src/styles/tokens.css` layout: base `:root {}` (light palette) → `:root[data-theme="dark"] {}` → density override → a **"redesign pass" override block at the very bottom** that silently re-declares `--surface-content`/`--surface-card`/`--surface-navigation`/`--shadow-card` for both themes. Always edit surfaces/shadows there too, not only in the top palette blocks, or the override wins and your edit appears to do nothing. Mica/backdrop compositing math itself is documented in `mem:frontend/synthetic_backdrop`.
- As of 2026-07-15 the palette was re-themed to "Signal": graphite ground, one reserved amber accent (`#c5590c` light / `#ff8a3d` dark), JetBrains Mono (`--font-family-mono`) for numeric/tabular data, Space Grotesk for headings. This changed token *values* only — the mica opacity/multiplier formula and the `tokens.css` block structure above are unchanged.
- **Accent color is NOT fully controlled by tokens.css.** `stores/navigation.svelte.ts::applyAppearance()` unconditionally overwrites `--accent-*`/`--text-on-accent` on `document.documentElement` via `createAccentPalette(systemAccent, theme)` (`lib/appearance/colors.ts`), deriving the accent from the user's live Windows system accent color. There is no user-facing toggle to disable this. The static `--accent-default` etc. in tokens.css are only the fallback used before the system accent is read (or under `forced-colors`). Don't "fix" an accent color mismatch by editing tokens.css alone — check this override first.
- The canonical app logo/mark is `frontend/src/assets/ravyn-mark.svg` (a self-contained square tile with its own baked-in dark background + feather glyph, not a monochrome/currentColor icon). It's imported directly as a Vite asset URL (`import ravynMark from "../../assets/ravyn-mark.svg"`) in `NavigationView.svelte` (sidebar brand mark) and `AboutSettings.svelte` (About hero) — do not reintroduce a text/letter placeholder mark in either spot. The same source is also copied to `frontend/public/favicon.svg` (browser/dev-preview tab icon) and rasterized into `src-tauri/icons/` (see `mem:desktop/core` for the native-icon rebuild gotcha).
- `lib/shell/CommandBar.svelte` is unused/dead code (no imports anywhere in `src/`) — the real per-page toolbar is `lib/components/PageCommandBar.svelte`, used via `PageScaffold`'s `commandBar` snippet. Don't restyle or extend the shell one expecting it to affect any visible screen.
- A `backup/pre-signal-ui` git branch (pointing at the pre-reskin commit) exists as a revert point for the 2026-07-15 visual reskin.
