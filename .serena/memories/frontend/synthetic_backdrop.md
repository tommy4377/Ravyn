# Synthetic backdrop (micablur) calibration

- The synthetic Mica pipeline (appearance.rs → `desktop_appearance` → `systemAppearance.svelte.ts` → `AppBackdrop.svelte`) works end to end; visibility is governed entirely by multiplied CSS layers in `frontend/src/styles/tokens.css`.
- Effective wallpaper visibility = wallpaper opacity (`--material-intensity` × 0.88 ≈ 0.67) × `--backdrop-brightness` × (1 − tint alpha) × (1 − surface alpha). All four multiply: individually plausible values compound to near-zero. Tuned 2026-07-15 so the product lands ≈ 0.2–0.3 behind content; verified by live screenshots against a colorful wallpaper.
- The "redesign pass" override block at the BOTTOM of tokens.css re-declares `--surface-content`/`--surface-card`/`--surface-navigation` and silently overrides the theme blocks above — edit surfaces there, not (only) in the top palette blocks.
- Debugging the webview: `tauri-plugin-mcp-bridge` JS tools need BOTH `withGlobalTauri: true` AND `mcp-bridge:default` in an ACTIVE capability. `tauri.conf.json` pins `security.capabilities` to `["setup-capability","main-capability"]`, so `capabilities/default.json` (which has mcp-bridge:default) is inert. Both changes are compile-time (rebuild required) and must be reverted after debugging.
- Bridge-less webview diagnostics that work: temporary `fetch()` beacons to a local Node listener (CSP connect-src allows `http://127.0.0.1:*`). `document.title` does NOT propagate to the native window title in Tauri, so it is useless as a signal.
- Debug builds embed the Vite `devUrl` (localhost:1420) — running `target/debug/ravyn-desktop.exe` without `npm run dev` yields an empty webview and a hidden 16×16 main window (frontend never calls `main_window_ready`).
- Wallpaper source is read from `HKCU\Control Panel\Desktop` and cached to `%LOCALAPPDATA%\app.ravyn.desktop\backdrop\` (only asset-protocol-scoped dir). Known gap (docs/SYNTHETIC_BACKDROP.md): slideshow/per-monitor setups may need an `IDesktopWallpaper` COM provider.
