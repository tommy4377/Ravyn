# Backend map
- `src/core`: manager orchestration, lifecycle, dispatch/execution, automation, bandwidth, events/progress/metrics, media/torrent control. Fair HTTP bandwidth is weighted/capped/floored, demand-aware (500 ms idle yield with synchronous rejoin), live-reconfigurable, and supports persistent non-overlapping IANA-time-zone windows reapplied by a supervised 15-second task.
- `src/download`: HTTP adapter, probe, adaptive planner, segmented range engine; persistent host profiles and fair bandwidth flows. Validated mirrors are admitted by exact length plus matching validator or checksum identity, then scheduled concurrently at non-overlapping work-unit boundaries; Metalink pieces are buffered up to 64 MiB, verified before commit, mirror-quarantined on corruption, and may use one delayed bounded hedge with loser-task shutdown. Single-stream SHA-256 is updated while bytes are written; resume rebuilds hash state only from the persisted prefix, and mismatches remove the partial before mirror retry. Non-piece segmented transfers use a byte-range `DurableCoverage` structure feeding one ordered whole-file SHA-256 task; idle workers split the tail of the largest active range (>=16 MiB remaining) into new durable units (`segments::split`, resume accepts any exact tiling), and one bounded identity-anchored speculative hedge per unit (<=64 MiB in isolated memory, try_acquire-only connections, slow-tail detection: stalled or projected >5 s while under half aggregate pace) completes stalled tails with loser cancellation. Split/hedge coordination is serialized through the per-transfer unit registry tokio Mutex; completion always syncs file bytes before the segment checkpoint.
- `src/adapters`: yt-dlp media and rqbit torrent integrations with typed wire normalization. All rqbit JSON bodies are streamed through a 4 MiB hard ceiling. The torrent run loop tolerates up to 10 consecutive stats-poll failures (covers rqbit restarts; tested against a mock engine).
- `src/services`: security, secrets, rules/schedules/cron, checksums, Metalink, managed engines, process supervision, browser/sniffer/import helpers, presets (explicit-wins merge), trust scoring (Ed25519 strict verification), library sub-modules (category, template, scan, trash, cleanup, root). Rule evaluation has a bounded read-only preview contract.
- `src/storage`: SQLx SQLite repository split by domain (21 modules). Audit entries form a transactionally serialized SHA-256 chain with retention anchors and `/v1/audit/verify`. Library entries use partial unique live-path index; basket uses atomic position allocation; profiles enforce single-active via partial unique index.
- `src/api`: Axum router with 7 route modules (automation, browser, jobs, library, media, system, torrents), bounded pagination, OpenAPI (128 unique method/path pairs). `POST /v1/rules/preview` (max 1,000 rules), `POST /v1/settings/validate` (per-field validation report, isolated blame, no persistence), and library/preset/basket/profile/trust/cleanup/statistics endpoints are read-only or audited contracts.
- `src/postprocess`: journaled post-transfer pipeline and named FFmpeg presets.
- Tests: colocated unit/property/fault/migration tests plus `tests/http_integration.rs`; 11 cargo-fuzz targets; Criterion policies in `benches/`. Portable fault fixtures cover read-only/locked partial files, DB-behind-file crash recovery, checksum shutdown, and rqbit restart. `MIN_WORK_UNIT_BYTES`/`PERSIST_INTERVAL_BYTES`/`HEDGE_MIN_REMAINING_BYTES` shrink under cfg(test) so in-crate tail tests stay small; integration tests see production values.
- Migrations: 20 ordered immutable files in `migrations/`; 29 tables total. Add migrations rather than rewriting released ones. Migrations 0019 (library) and 0020 (presets/profiles/basket) are additive-only.
- Long-running work must be cancellation-aware; DB/file state must remain consistent and late completion must not overwrite paused/cancelled state.
- Preserve loopback defaults, output-root confinement, private-network blocking, bounded inputs, and least privilege.
- rqbit response/capability invariants: `mem:backend/rqbit_contracts`.
- Runtime settings classification and live reconfiguration: `mem:backend/runtime_settings`. The `new_jobs_only` classification needs a config-snapshot refactor (54 `self.config.` sites across 7 structs) — see report.md.
- yt-dlp process parsing and playlist lifecycle invariants: `mem:backend/media_lifecycle`.
- Library features, presets, basket, profiles, trust, cleanup: `mem:backend/library`.
